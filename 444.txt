top(динамический мониторинг процессов):
a) top -d 5 - время обновления 5 сек
b) top -u ksenia - только процессы user ksenia
c) top p 1024, 2048, 4096 -мониторинг определённого PID
d) top -n 10 - вывод процессов с максимумом 10 итерациями
e) top -b -n 12000 > file_name - тоже что и в d), но перенаправит поток вывода в файл

top
1)<Shift+N> - сортировка по PID
2)<Shift+P> - сортировать по использованию СPU
3)<Shift+M> - сортировать по использованию памяти
4)<Shift+T> - сортировка по времени выполнения
5)d -  изменение задержки
6)= - дисплей задачи текущего окна быть виден
7)u - сортировать по имени пользователя
8)n - изменить число отображаемых процессов
9)K - убить процесс

/proc:
Можно найти PID информацию о процессах
вся информация из /proc в top

пр: cat /proc/loadavg - нагрузка на систему за последние 1, 5, 15 минут
/proc/cpuinfo - информация о процессе во время загрузки
откуда инфа - /proc/PID/cwd

kill -l просмотреть все сигналы

-1 (для su)


-s signal_name
Символическое имя сигнала, задающее сигнал для отправки сигнала не по умолчанию.

-l [exit_status]
Если операнд не указан, то показать имена сигналов; В противном случае, написать название сигнала, соответствующего exit_status.

-signal_name
Символическое имя сигнала, задающее сигнал для отправки в TERM по умолчанию.0

-signal_number
Неотрицательное десятичное целое, задающее сигнал для отправки в TERM по умолчанию.
